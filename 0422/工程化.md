###  工程化基础与单一职责
1. 分层的目录和文件的叫法，不是一种编程习惯。
 - 分层设计思想
 - 单一职责思想
 - 路由分发思想
 - MVC思想
 - 模块化思想
 - 配置与执行分离思想

1. 入口文件有什么？
 - 接口请求（接收请求）
 - 判断method
 - 判断url
 - 接受参数 | post / get
 - 读取数据
 - 查询数据
 - 组装数据
 - 返回结果

 * 前端也不能写在一个文件中的
    - 写页面的结构
    - 发送请求
    - 处理接收数据
    - 拼接HTML
    - 绑定事件
    - 动态逻辑

2. 项目越大出现的问题就会显现
 - 太长，不好读
 - 修改一个地方的代码，影响到了另外的代码
 - 别人接手去维护，他就疯了
 - 重复的逻辑很多，造成了代码的冗余

3. 单一职责
 - 一个模块就干一件事
```js

    function setNumberToString(num) {
        return String(num)
    }

    if (setNumberToString(res.code) === '200')

```

- routes 只负责请求往哪里走
- controller 只负责如何去处理这个请求
- data 只负责提供数据
- index 只负责启动服务
- 综上所述就是单一职责



### 分层设计
1. 后端
    - 入口
    - 路由
    - 控制层
    - 数据层

 * 请求怎么进来是一个问题
 * 请求交给谁是一个问题
 * 数据怎么读是一个问题
 * 返回的数据是什么是一个问题

2. 前端
    - 页面
    - 组件
    - 服务层
    - 工具层

* 当你知道了为什么要分层，以及如何分层，分为哪些层的时候。你的问题可能是文件夹为什么叫这个名字？
* 文件夹命名并不是某一个程序员的习惯，这个是开发社区经过时间累加的结果。
